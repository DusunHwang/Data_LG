## 1. 문서 범위

이 문서는 현재 버전에서 사용자가 실행하는 주요 분석 기능이 어떤 기준으로 동작하는지 설명한다. 대상 기능은 다음과 같다.

- Subset clustering: 결측 구조와 컬럼 특성을 기준으로 모델링에 유리한 부분 데이터셋을 찾는 기능
- 핵심인자 추출: 모델, SHAP, Null Importance, 단순화 모델을 이용해 중요한 입력 인자를 판정하는 기능
- 모델링: LightGBM 회귀 모델과 Decision Tree 임계값 분류 모델
- 최적화 위저드: Null Importance, 피처 선택, LGBM/BCM 모델 기반 역최적화, 고정 횟수/고정 시간 실행, 수렴 중단

이 문서는 사용 관점의 알고리즘과 판정 기준만 다룬다. 내부 라우팅, 인텐트 분류, API, DB 저장 구조는 개발자 버전에 정리되어 있다.

## 2. 기본 전제

분석은 현재 선택된 데이터셋 또는 사용자가 선택한 데이터프레임 아티팩트를 기준으로 수행된다.

모델링, SHAP, 최적화처럼 타겟이 필요한 기능은 다음 순서로 타겟 컬럼을 찾는다.

1. UI에서 선택된 타겟 컬럼
2. 현재 브랜치나 데이터셋 설정에 저장된 타겟 컬럼
3. 사용자가 입력한 문장에 포함된 컬럼명

예를 들어 `Decision Tree 로 young_modulus_calc_GPa 150 이상과 이하의 분류 모델을 만들어줘`처럼 문장 안에 컬럼명과 기준값이 있으면, UI에서 타겟을 따로 지정하지 않아도 해당 문장을 분석해 분류 타겟을 구성한다.

## 3. Subset Clustering

Subset clustering은 일반적인 거리 기반 클러스터링이 아니라, 결측 구조와 컬럼 품질을 기준으로 사용할 만한 부분 데이터셋을 찾는 절차이다. 목적은 다음과 같다.

- 결측이 너무 많거나 정보량이 낮은 컬럼을 제외한다.
- 비슷한 결측 패턴을 가진 행들을 묶는다.
- 특정 범주값에서 데이터가 더 조밀하게 존재하는 부분집합을 찾는다.
- 전체 데이터와 거의 같은 후보는 별도 subset으로 만들지 않는다.

### 3.1 컬럼 분류 기준

각 컬럼은 먼저 아래 기준으로 분류된다.

| 분류 | 기준 | 처리 의미 |
| --- | --- | --- |
| target | 사용자가 지정했거나 추론된 타겟 컬럼 | 모델링/최적화 기준 컬럼으로 보존 |
| high_missing | 결측률이 80% 초과 | subset 후보 생성에서 제외 |
| constant | 고유값이 1개 이하 | 정보량이 없어 제외 |
| near_constant | 고유값 비율이 0.5% 미만 | 정보량이 낮아 제외 |
| id_like | 비수치형이고 고유값 비율이 95% 초과 | ID 성격으로 보고 제외 |
| low_cardinality | 범주형이고 고유값 수가 20개 미만 | 층화 subset 후보 생성에 사용 |
| numeric | 일반 수치형 컬럼 | 모델링/최적화 후보 피처 |
| categorical | 일반 범주형 컬럼 | 모델링 후보 피처 |

타겟 컬럼은 다른 제외 조건보다 우선해서 보존된다.

### 3.2 결측 구조 분석

컬럼 분류 후에는 각 행이 어떤 컬럼에서 결측인지 분석한다.

- 행 결측 서명: 한 행에서 결측인 컬럼들의 조합
- 공동 결측: 두 컬럼이 함께 결측되는 빈도
- 분석 대상: 상수, 준상수, ID형 컬럼을 제외한 컬럼

이 분석 결과는 subset 후보 생성의 기반이 된다.

### 3.3 Subset 후보 생성 방식

현재 버전은 네 가지 방식으로 후보를 만든다.

| 후보 방식 | 생성 기준 |
| --- | --- |
| row signature | 같은 결측 패턴을 가진 행을 묶음 |
| low-cardinality stratification | 값 종류가 적은 범주형 컬럼의 주요 값별로 행을 묶음 |
| hybrid dense | 행 결측률과 컬럼 결측률이 낮은 조밀한 부분을 선택 |
| complete cases | 사용 가능한 주요 컬럼이 모두 결측이 아닌 행을 선택 |

너무 작은 후보는 제외된다. 일반적으로 행 수가 10개 또는 20개 미만인 후보는 방식에 따라 제외된다.

### 3.4 Subset 점수

각 subset 후보는 다음 요소를 곱해 점수를 계산한다.

```text
score = row_coverage * feature_coverage * (1 - mean_missingness) * target_completeness
```

각 요소의 의미는 다음과 같다.

| 요소 | 의미 |
| --- | --- |
| row_coverage | 전체 행 중 subset에 포함된 행 비율 |
| feature_coverage | 전체 컬럼 중 subset에 포함된 컬럼 비율 |
| mean_missingness | subset 내부 평균 결측률 |
| target_completeness | 타겟 컬럼의 비결측 비율 |

점수가 높을수록 행과 피처를 충분히 보존하면서 결측이 적고 타겟이 잘 채워진 subset이다.

### 3.5 전체 데이터와 거의 같은 후보 제거

다음 조건을 동시에 만족하는 후보는 전체 데이터와 사실상 같다고 보고 제외한다.

- 행 커버리지 95% 이상
- 피처 커버리지 95% 이상

의미 있는 후보가 남지 않으면, 시스템은 별도 subset을 만들지 않고 전체 데이터로 모델링하는 것을 권장한다.

### 3.6 출력물

Subset clustering은 다음 결과물을 만든다.

- 컬럼 분류표
- 결측 구조 리포트
- subset registry
- subset 점수표
- 각 subset 데이터프레임
- 결측 heatmap
- subset 요약 리포트

## 4. 모델링과 핵심인자 추출

현재 버전의 핵심인자 추출은 하나의 방법만 쓰지 않는다. 분석 목적에 따라 다음 방법을 조합한다.

- LightGBM 회귀 모델의 feature importance
- Decision Tree 분류 모델의 feature importance
- SHAP 중요도
- Null Importance
- 단순화 모델의 성능 비교

## 5. LightGBM 회귀 모델링

연속형 타겟을 예측할 때는 LightGBM 회귀 모델을 기본 모델로 사용한다.

### 5.1 학습 데이터 준비

모델링 전에 다음 처리가 수행된다.

- 타겟 결측 행은 제외한다.
- 타겟은 수치형이어야 한다.
- 타겟 고유값이 1개 이하이면 모델링하지 않는다.
- 전체 유효 행이 너무 적으면 모델링하지 않는다.
- 상수 컬럼과 ID 성격의 비수치형 컬럼은 피처에서 제외한다.
- 범주형 컬럼은 결측을 `__missing__`으로 채운 뒤 범주형으로 처리한다.

### 5.2 모델 학습 기준

기본 학습 방식은 다음과 같다.

| 항목 | 기준 |
| --- | --- |
| 모델 | LightGBM regression |
| 검증 분할 | train 80% / validation 20% |
| random_state | 42 |
| 최대 boosting round | 200 |
| early stopping | 30 round |
| 주요 지표 | RMSE, MAE, MAPE, R2 |

Champion 모델은 validation RMSE가 가장 낮은 모델이다.

### 5.3 결과 해석

- RMSE가 낮을수록 예측 오차가 작다.
- R2가 높을수록 타겟 변동을 더 잘 설명한다.
- feature importance는 모델이 예측에 많이 사용한 입력 인자를 보여준다.
- feature importance는 인과관계를 의미하지 않는다.

## 6. Decision Tree 임계값 분류

사용자가 문장에 `150 이상/이하`, `분류 모델`, `Decision Tree`처럼 기준값 기반 분류를 명시하면 Decision Tree 분류 모델을 만든다.

예:

```text
Decision Tree 로 young_modulus_calc_GPa 150 이상과 이하의 분류 모델을 만들어서 어떤 인자가 가장 중요한 인자인지 찾아줘
```

이 경우 시스템은 다음 이진 타겟을 자동 생성한다.

```text
young_modulus_calc_GPa >= 150
```

### 6.1 적용 조건

Decision Tree 임계값 분류는 다음 조건을 만족해야 실행된다.

- 요청 문장에 실제 컬럼명이 포함되어 있다.
- 해당 컬럼이 데이터셋에 존재한다.
- 해당 컬럼이 수치형이다.
- 기준값이 숫자로 해석된다.
- 기준값 기준으로 양쪽 클래스가 모두 존재한다.
- 유효 행이 최소 20개 이상이다.

### 6.2 모델 기준

| 항목 | 기준 |
| --- | --- |
| 모델 | DecisionTreeClassifier |
| max_depth | 4 |
| min_samples_leaf | `max(2, train 행 수의 2%)` |
| class_weight | balanced |
| 검증 분할 | train 80% / validation 20% |
| random_state | 42 |
| 지표 | accuracy, precision, recall, f1 |

수치형 피처는 중앙값으로 결측을 채운다. 범주형 피처는 `__missing__`으로 결측을 채운 뒤 one-hot encoding한다.

### 6.3 중요 인자 판정

Decision Tree의 feature importance를 사용한다. 범주형 피처가 one-hot encoding으로 여러 컬럼으로 나뉜 경우, 중요도를 원래 컬럼명 기준으로 다시 합산한다.

가장 중요한 인자는 합산 importance가 가장 큰 원본 컬럼이다.

## 7. SHAP 기반 핵심인자 추출

이미 학습된 champion 모델이 있을 때 SHAP 분석을 수행할 수 있다.

### 7.1 SHAP 계산 기준

- champion 모델을 불러온다.
- 현재 데이터셋에서 타겟 결측 행을 제외한다.
- 학습 당시 피처 순서와 전처리 규칙을 맞춘다.
- 최대 5,000행까지 샘플링해 SHAP 값을 계산한다.
- TreeExplainer를 사용한다.
- 각 피처의 중요도는 `mean(abs(SHAP value))`로 계산한다.

### 7.2 해석

SHAP 중요도는 모델 예측값을 움직이는 평균 영향력을 나타낸다.

- 값이 클수록 모델 예측에 미치는 영향이 크다.
- 양/음 방향은 개별 SHAP plot에서 해석한다.
- 평균 절대값 순위는 전역 중요도 순위이다.

## 8. 단순화 모델

핵심 피처만으로 모델을 줄일 수 있는지 확인하기 위해 단순화 모델을 학습한다.

### 8.1 후보 피처 수

현재 후보 피처 수는 다음과 같다.

```text
top 3, top 5, top 8, top 12
```

SHAP 중요도 순위 상위 k개 피처만 사용해 LightGBM 모델을 다시 학습한다.

### 8.2 채택 기준

기준 모델 대비 RMSE 증가가 10% 이내이면 성능 저하가 허용 가능한 단순화 모델로 본다.

```text
drop_ratio = simplified_rmse / baseline_rmse
허용 기준: drop_ratio <= 1.1
```

허용 가능한 후보가 여러 개이면, 가장 적은 피처를 쓰는 모델을 추천한다. 기준을 만족하는 후보가 없으면 전체 모델 유지를 권장한다.

## 9. 최적화 위저드

최적화 위저드는 모델 예측값을 최대화 또는 최소화하는 입력 조건을 찾는 절차이다.

현재 프로세스는 다음 순서로 진행된다.

1. 사용할 데이터셋 또는 subset 선택
2. 최적화 대상 타겟 선택
3. 필요한 champion 모델 확인
4. Null Importance 기반 추천 피처 계산
5. 사용자가 최적화 피처, 고정값, 조성 제약을 조정
6. 목표, 방향, 제약 조건, 모델 타입 선택
7. BCM 선택 시 최적화 전에 BCM 모델 선학습
8. 고정 횟수 또는 고정 시간 실행 방식 선택
9. 최적화 실행
10. 수렴, 시간 제한, 반복 제한, 취소 중 하나의 사유로 종료
11. 최적 조건과 예측값 표시

## 10. Null Importance

최적화에서 사용할 핵심 인자를 추천하기 위해 Null Importance를 계산한다.

### 10.1 실제 중요도

먼저 champion 모델에 대해 SHAP 중요도를 계산한다.

```text
actual_importance = mean(abs(SHAP value))
```

### 10.2 Null 중요도

타겟 값을 무작위로 섞은 뒤 모델을 여러 번 다시 학습한다. 현재 기본 반복 수는 30회이다.

각 반복에서 SHAP 중요도를 계산하면, 우연히 나타날 수 있는 중요도 분포가 만들어진다. 이 분포를 Null 분포라고 한다.

### 10.3 유의 피처 판정

현재 기준은 다음과 같다.

```text
actual_importance > null_importance의 90 percentile
```

즉 실제 중요도가 무작위 타겟으로 만든 중요도 분포의 상위 10% 기준보다 크면 유의한 피처로 본다.

추천 피처가 너무 적으면 실제 중요도 순위로 보충하고, 최대 15개까지 추천한다.

### 10.4 다중 타겟 추천

여러 타겟을 동시에 고려할 때는 각 피처가 몇 개 타겟에서 중요하게 나타났는지와 각 타겟 내 중요도 점수를 함께 본다.

정렬 기준은 다음과 같다.

1. 더 많은 타겟에서 선택된 피처 우선
2. 집계 점수가 높은 피처 우선
3. 이름순

집계 점수는 coverage와 중요도 점수를 함께 반영한다.

## 11. 최적화 모델 타입

### 11.1 LGBM

LGBM을 선택하면 기존 champion LightGBM 모델의 예측값을 목적함수로 사용한다. 학습된 모델을 그대로 사용하므로 최적화 단계에서 별도 모델 학습은 하지 않는다.

### 11.2 BCM

BCM을 선택하면 최적화 전에 BCM 모델을 먼저 학습한다. 이 단계가 끝난 뒤 사용자는 고정 횟수 또는 고정 시간을 선택하고 최적화를 시작한다.

현재 BCM은 다음 모델을 결합한다.

- 기존 LightGBM champion 모델
- RBF kernel Gaussian Process expert
- DotProduct kernel Gaussian Process expert

Gaussian Process는 표준화된 수치 피처를 사용한다. 데이터가 너무 크면 내부 기준에 따라 최대 행 수를 제한한다.

BCM 예측은 두 Gaussian Process expert의 평균과 분산을 결합한 뒤, 최종적으로 LightGBM 예측과 50:50으로 섞는다.

```text
final_prediction = 0.5 * BCM_prediction + 0.5 * LGBM_prediction
```

따라서 BCM 선택 시 최적화 실행 단계는 이미 학습된 BCM 모델을 불러와 최적화만 수행한다.

## 12. 최적화 실행 방식

최적화는 `differential_evolution` 방식으로 수행된다. 이는 여러 후보 입력값을 반복적으로 생성하고, 모델 예측값이 좋아지는 방향으로 탐색하는 전역 최적화 방법이다.

### 12.1 기준 행

최적화 시작점에 해당하는 기준 행은 다음 방식으로 만든다.

- 수치형 피처: 중앙값
- 범주형 피처: 최빈값
- 모델에 필요한데 데이터에 없는 피처: 0
- 사용자가 고정한 값: 사용자 입력값 우선

### 12.2 최적화 대상 피처

최적화 대상 피처는 다음 조건을 통과한 피처이다.

- Null Importance 추천 피처 또는 사용자가 선택한 피처
- 현재 데이터셋 또는 subset에 존재하는 피처
- 범주형이 아닌 피처
- 고정값으로 지정되지 않은 피처
- 조성 제약의 balance 피처가 아닌 피처

고정값으로 지정된 피처는 최적화 중 바뀌지 않는다.

### 12.3 탐색 범위

각 피처의 탐색 범위는 다음 순서로 결정된다.

1. 사용자가 지정한 범위
2. 데이터에서 관측된 최소/최대값
3. 둘 다 없으면 0에서 1

사용자가 확장 비율을 지정하면 데이터 범위를 바깥으로 확장해 탐색한다.

## 13. 제약 조건

### 13.1 타겟 제약

여러 예측 타겟에 대해 최소값/최대값 제약을 걸 수 있다. 제약 위반은 목적함수에 큰 패널티로 반영된다.

```text
penalty = 1,000,000 * violation
```

### 13.2 조성 합계 제약

조성 합계 제약은 여러 조성 컬럼의 합이 지정된 total이 되도록 하는 조건이다.

balance 피처는 직접 최적화하지 않고 다음 식으로 자동 계산된다.

```text
balance_feature = total - sum(other_composition_features)
```

조성 합계가 total에서 벗어나거나 각 조성값이 허용 min/max 범위를 벗어나면 패널티가 부여된다.

## 14. 고정 횟수, 고정 시간, 수렴 중단

사용자는 최적화를 고정 횟수 또는 고정 시간으로 실행할 수 있다.

### 14.1 고정 횟수

고정 횟수 모드는 지정한 평가 횟수에 도달할 때까지 탐색한다. 단, 중간에 수렴하면 지정 횟수를 모두 채우지 않고 종료한다.

### 14.2 고정 시간

고정 시간 모드는 지정한 시간까지 탐색한다. 단, 중간에 수렴하면 남은 시간이 있어도 종료한다.

### 14.3 수렴 기준

현재 수렴 기준은 다음과 같다.

```text
최근 8세대 동안 누적 최적값 개선량 <= max(1e-6, 이전 최적값의 절대값 * 1e-6)
```

최대화에서는 최적값이 얼마나 증가했는지, 최소화에서는 최적값이 얼마나 감소했는지를 본다. 개선량이 기준 이하이면 더 탐색해도 의미 있는 개선이 어렵다고 보고 수렴으로 종료한다.

### 14.4 종료 사유

결과에는 종료 사유가 함께 표시된다.

| 종료 사유 | 의미 |
| --- | --- |
| converged | 사용자 지정 횟수나 시간 전에 수렴 기준을 만족 |
| time_limit | 지정 시간이 먼저 도달 |
| max_iterations | 지정 반복/평가 한도에 도달 |
| scipy_converged | 최적화 엔진 자체 수렴 조건 만족 |
| cancelled | 사용자가 취소 |

수렴으로 종료되더라도 그 시점까지 발견한 최적 조건과 최적 예측값을 보여준다.

## 15. 단순 최대값 질의

사용자가 `최대값 알려줘`, `컬럼 A의 max는?`처럼 단순 통계값을 물으면 현재 버전은 그래프를 만들지 않고 스칼라 집계를 우선 수행한다.

지원 연산은 다음과 같다.

- max
- min
- mean
- sum
- count
- median

단, `최대화`, `최적화`처럼 모델 기반 최적화를 의미하는 표현은 단순 max 집계로 처리하지 않는다.

## 16. 결과를 해석할 때 주의할 점

- 중요 인자는 모델이 예측에 사용한 패턴의 강도를 뜻하며, 인과관계 자체를 증명하지 않는다.
- subset 점수가 높다는 것은 해당 subset이 결측과 커버리지 측면에서 유리하다는 뜻이지, 반드시 물리적으로 다른 군집이라는 뜻은 아니다.
- 최적화 결과는 모델 예측값을 기준으로 한 추천 조건이다. 실제 실험값을 보장하지 않는다.
- 데이터 범위 밖으로 탐색 범위를 확장하면 더 높은 예측값을 찾을 수 있지만, 외삽 위험도 커진다.
- BCM은 Gaussian Process expert를 추가로 사용하므로 LGBM보다 계산 비용이 크다. 현재 프로세스에서는 이 비용을 최적화 전에 선학습 단계로 분리한다.
