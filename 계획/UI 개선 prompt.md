## Target Dataframe 설정
Target dataframe은 분석의 대상이 되는 데이터 프레임을 뜻한다.
Target dataframe 은 기본적으로 매번생성된 채팅영역의 맨 마지막 데이터프레임이 된다.
Target dataframe 은 현재 구현된 아티펙트 패널에 나타나는 분석 대상 데이터 프레임처럼 다른 데이터 프레임과 구별되는 테두리 색을 지니고 있으며, 타겟 설정 버튼이 현재 아티팩트 패널처럼 존재한다.
타겟 설정 버튼을 누르면 아티팩트 패널에서의 동작 처럼 전체 컬럼이름이 아이콘으로 표시되고, 클릭하여 설정하고 완료를 누르면 최종 설정 된다.
변수 설정 버튼을 누르면 동일하게 컬럼이름이 아이콘으로 모두 나타나고 디폴트로 모두 설정 되어있고, 타겟은 회색으로 나타나서 선택이 불가능하다. 선택된 컬럼이름 아이콘은 클릭하면 select/deselect 가 토글되고, 완료 버튼을 누르면 최종 설정된다.
데이터 프레임으로 
각 데이터 프레임에는 “이 데이터에 요청” 버튼이 있고, 이 버튼을 누르면 해당 데이터 프레임이 target dataframe 이 된다.
해당 데이터 프레임을 대상으로 모델링을 포함한 요청이 들어오면 이 변수와 타겟을 전달하여 모델링을 수행한다.


## 사이드바 변경
### Question Templates
브랜치별로 분석 프레임을 고정하는 워크플로우는 제거한다.
왼쪽 사이드바에 나타나는 브랜치 바로가기 및 관련 기능은 제거한다.
왼쪽 사이드바에는 루트의 QuestionList.md 파일에 정의된 질문리스트를 나열하고 각 질문은 클릭하면 대화입력창에 입력만 되어 편집을 기다린다.
질문리스트의 구조는 #, ## 등의 title로 hierarchy를 구성해서 접었다 폈다 할 수 있고, * 로 표시된 내용이 클릭 가능한 하나의 문장이 된다.


### Top-down 아티팩트 graph

오른쪽의 아티팩트 패널은 채팅이력의 top/down graph 구조를 나타내는 history shortcut 으로 대체 된다.
history shortcut은 mermaid 형태의 TD graph로 연결이되고, 위에서 아래로 진행된다.
데이터프레임, image(plot), Question 등이 적당한 아이콘으로 표시되고 옆에는 간략한 10자 정도의 설명이나 질문 요약이 나타난다. image는 썸네일이 아이콘이 된다. 데이터프레임에서 나온 화살표는 질문으로 이어지고 질문에서는 도출된 결과물들의 아이콘이 표시된다. 각 아이콘은 클릭하면 채팅패널은  해당 아티팩트가 존재하는 위치로 스크롤링되어 이동 된다.
